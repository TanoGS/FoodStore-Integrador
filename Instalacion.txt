backend:
1. Requisito previo: tener Python 3.11+ instalado y en el PATH.
   Cualquiera de estas opciones funciona:
     a) Python oficial (recomendado): https://www.python.org/downloads/
     b) Miniconda:                    https://docs.conda.io/en/latest/miniconda.html
     c) Anaconda:                     https://www.anaconda.com/download
     d) pyenv-win (avanzado):         https://pyenv-win.github.io/

   Verificación rápida en PowerShell:
     python --version
     python -c "import sys; print(sys.platform)"   # debe decir "win32"

   ⚠️ EVITAR Pythons en ubicaciones no estándar (ej: D:\bin\python.exe):
     pueden crear venvs con estructura Unix (bin/) que NO funcionan en Windows.

2. Crear y activar el entorno virtual:
python -m venv venv

# En Windows (PowerShell):
      .\venv\Scripts\Activate.ps1

# En Windows (CMD):
venv\Scripts\activate.bat

3. Instalar las dependencias:
pip install -r requirements.txt

Fase 1: Sacar la copia (Exportar en tu PC actual)
Abre pgAdmin en tu computadora actual y conéctate a tu servidor local.

Despliega la lista de bases de datos y busca la tuya (ej. food_store_db).

Haz clic derecho sobre el nombre de tu base de datos y selecciona Backup... (Copia de seguridad).

Se abrirá una ventana. En la pestaña General:

Filename (Nombre de archivo): Haz clic en el ícono de la carpeta, elige dónde guardarlo 
(por ejemplo, en tu Escritorio) y ponle un nombre como backup_food_store.sql o backup_food_store.tar.

Format (Formato): Selecciona Custom (Personalizado) o Plain (Texto plano). Te recomiendo Custom porque es más fácil de restaurar.

Fase 2: Cargar la copia (Importar en la PC nueva)
Una vez que estés en la computadora nueva y hayas instalado PostgreSQL y pgAdmin (como vimos en la guía anterior):

Abre pgAdmin en la PC nueva.

Crea una base de datos vacía: Haz clic derecho en Databases -> Create -> Database... y llámala exactamente igual (food_store_db).

Haz clic derecho sobre esa base de datos vacía que acabas de crear y selecciona Restore... (Restaurar).

En la pestaña General:

Format: Selecciona el mismo que usaste (Custom).

Filename: archivo backup_food_store  

Haz clic en el botón Restore.
